Import ELF with Namespace

Location: CodeBrowser → File → Import ELF with Namespace…

Overview

This action imports an ELF binary into the current Ghidra project while assigning all symbols to user-defined namespaces. Using separate namespaces for the existing binary and the newly imported binary prevents symbol-name clashes when both share common names (e.g. main, init).

Usage

  1. Open a program in the CodeBrowser.
  2. Select File → Import ELF with Namespace…
  3. In the dialog that appears:
    • Click Browse… to select the .elf file to import.
    • Enter a Existing Binary Namespace. All global symbols currently in the open program will be moved into this namespace. The default value is the program name with its file extension removed.
    • Enter a New Binary Namespace. All global symbols in the imported binary will be placed into this namespace. The default value is the selected file name with the .elf extension removed.
  4. Click OK to start the import.

The operation runs in the background and reports progress in Ghidra's task dialog. It can be cancelled at any time.

What Happens

  1. A namespace with the specified existing-binary name is created in the currently open program (if it does not already exist). Every global symbol in that program is then moved into this namespace.
  2. The selected ELF file is imported into the root folder of the current project using Ghidra's standard automatic loader.
  3. A namespace with the specified new-binary name is created in the imported program. Every global symbol of the imported program is moved into this namespace.

Notes

  • Symbols that cannot be moved (e.g., because a symbol with the same name already exists in the target namespace) are skipped; a warning is written to the Ghidra application log.
  • The two namespace names must be different.
  • The action is only available when a program is open in the CodeBrowser.
